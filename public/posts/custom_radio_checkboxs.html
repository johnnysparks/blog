CSS, unlike forms, is awesome. By summoning the mighty power of css selectors, we can move checkboxes and radio buttons out of the 90s. 

  <form id="styled"> 
    <input type="radio" name="emotion" id="happy" />
    <label for="happy">
      <img alt="happy" src="http://i.imgur.com/zm8fp.png" />
      <p>happy</p>
    </label>

    <input type="radio" name="emotion" id="sad" />
    <label for="sad">
      <img alt="sad" src="http://i.imgur.com/sKhy9.png" />
      <p>sad</p>
    </label>
  </form>

Which looks like:

How are you feeling today?

Cool, so that's the markup. Pretty basic, just wrapping the label around the image to get the hit area.

When I first wanted to sexy-up some form components, I planned to just jquery-whiz-bangify-modularize a pluginable formifier. But some well-placed CSS selectors do all the work for us. Incoming magic:

  <style>
    #styled input {
      display: none;
    }
    #styled label {
      display: inline-block;
    }
    #styled label p{
      text-align: center;
    }
    #styled input + label > img{
      border: 10px transparent solid;
    }
    #styled input:checked + label > img{
      border: 10px #58fe77 solid;
      -webkit-border-radius: 30px;
      -moz-border-radius: 30px;
      border-radius: 30px;
    }
  </style>

Which becomes:

How are you feeling today?

So that's just a radio button, but that style markup will work for checkboxes too, so long as your input ids are matching the label "for" attributes. Watch for those unique id's! I think chrome and firefox just match the first element id found, but IE7 will actually tell you to piss off. Which is frankly, easier to debug. That's right, I said it, IE7 is easier to debug!
